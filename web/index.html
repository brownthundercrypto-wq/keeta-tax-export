Keeta Tax Export

Turn your Keeta (KTA) transaction history into a CSV you can import into CoinLedger. Runs entirely in your browser.

v0.2.0. Tested against Keeta mainnet and @keetanetwork/keetanet-client 0.18.3 on 19 July 2026. If you are reading this much later, check that the SDK has not moved.

This only accepts a public address. It's the same one you would give someone so they can send you KTA. It never asks for a seed phrase or private key, and cannot use one. Nothing you type here is sent to us. There is no server.

Before you import, pick the right KTA

CoinLedger will ask you to choose between KTA - Keeta and KTA - KTA.

Choose KTA - Keeta.

The other one is an unrelated coin. Pick it and CoinLedger prices your entire report against the wrong asset. No error, no warning. CoinLedger can remember the choice, so you only make it once.

How to import

  1. In CoinLedger, go to Import → Other Account and upload the CSV.
  2. When it asks about shared tickers, choose KTA - Keeta.
  3. Read the review notes below and check those rows before you file.

About bridge transfers

If you moved KTA to another chain, that shows up on Keeta as an ordinary send. It looks exactly like a sale. We flag the bridges we can recognize, but we cannot detect all of them. Only two bridge operators publish their addresses in a way we can read.

If you bridged KTA to Base or anywhere else, check those rows yourself. A bridge you moved between your own wallets is usually not a taxable sale, but this file cannot tell the difference on its own.

How this works, and what it never sees

You paste a public address. Your browser talks straight to Keeta's own nodes, reads your transaction history, and builds the CSV locally. The file is created in your browser and saved by your browser.

There is no server in the middle. We do not run a backend, we do not have a database, and we never see your address. The only network requests this page makes are to Keeta's nodes and to load the Keeta SDK. You can verify that yourself in your browser's developer tools, Network tab.

It also never asks for a seed phrase or private key. It could not use one if you gave it one. Reading history only needs a public address.

There is nothing to monetize here. No ads, no tracking, no analytics, no affiliate links, no server. This page makes no money and collects nothing. That is the simplest reason to believe the rest of it.

Which tokens are included

This version exports every Keeta token whose decimal precision is published on-chain, which is almost all of them. Amounts are read from each token's own on-chain metadata rather than from a list we maintain, so a token we have never seen still converts correctly.

Swaps are exported as single trade rows, with the asset you gave up and the asset you received on one line. Routing fees paid in the same transaction are already folded into those figures.

Two things are still left out, on purpose

Tokenized fiat. Keeta has tokens whose tickers are currency codes: USD, EUR, JPY and others. Their amounts are correct and verified. The problem is the label. Tax software is very likely to read a ticker of USD as actual US dollars, and that produces a right number attached to the wrong asset. That is harder to catch than an obviously broken figure, so these are listed in your review notes instead of exported.

Any token whose divisor cannot be established. If a token publishes no decimal precision, or publishes something that disagrees with our reference list, the tool refuses that row rather than guessing. Guessing wrong is a factor-of-ten error in a tax return.

A warning about tokens nobody prices

Exporting a token is not the same as your tax software knowing what it was worth. CoinLedger only has price history for assets it already tracks. For anything else you have to add a custom asset and enter the price yourself, per transaction, at each date.

The tool tells you how many rows this affects before you download, so you can judge the work involved rather than discovering it afterwards.

What this does not do

This builds an import file. It does not calculate your cost basis, and it does not give you a finished tax return.

One Keeta address cannot tell you three things you need:

  • What you paid. The chain records that KTA moved, not what it cost you.
  • Which wallets are yours. Moving KTA between your own addresses is not a sale, but on-chain it looks the same as one.
  • Anything that happened off Keeta. Exchange buys, other chains, transfers in from somewhere else. None of it is here.

Your tax software fills those gaps once you import the rest of your accounts. That is the job this file feeds into, not the job it finishes.
